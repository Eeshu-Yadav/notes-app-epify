# notes-app — backend

Django 5 + DRF + SimpleJWT REST API for the Notes App assignment.

**Live**: https://notes-app-76mt.onrender.com — try `/about`, `/openapi.json`, `/docs/`.

## Setup

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env          # adjust values
python manage.py migrate
python manage.py runserver
```

Server runs at `http://127.0.0.1:8000`.

## Environment variables

| Variable | Purpose |
|---|---|
| `DJANGO_SECRET_KEY` | Django secret. Required. |
| `DJANGO_DEBUG` | `True` / `False`. Defaults to `False`. |
| `DJANGO_ALLOWED_HOSTS` | Comma-separated hostnames. |
| `DJANGO_CSRF_TRUSTED_ORIGINS` | Comma-separated origins for CSRF. |
| `DATABASE_URL` | Postgres URL in prod. Falls back to SQLite locally. |
| `CORS_ALLOW_ALL` | `True` to allow any origin (dev only). |
| `CORS_ALLOWED_ORIGINS` | Comma-separated allowed origins. |

## API endpoints

All endpoints live at the root. Auth uses `Authorization: Bearer <access>`.

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/register` | no | Create user, returns access + refresh tokens. |
| POST | `/login` | no | Obtain access + refresh tokens. |
| GET | `/me` | yes | Current user profile. |
| GET | `/notes` | yes | List notes owned by or shared with the user. |
| POST | `/notes` | yes | Create note. |
| GET | `/notes/{id}` | yes | Retrieve a single note. |
| PUT | `/notes/{id}` | yes | Replace a note. |
| PATCH | `/notes/{id}` | yes | Partial update. |
| DELETE | `/notes/{id}` | yes | Delete a note. |
| POST | `/notes/{id}/share` | yes | Share with another user by username. |
| POST | `/notes/{id}/unshare` | yes | Revoke a share. |
| POST | `/notes/{id}/public-link` | yes | Create or rotate a public share token. |
| DELETE | `/notes/{id}/public-link` | yes | Disable the public link. |
| POST | `/notes/{id}/toggle-pin` | yes | Toggle pin state. |
| GET | `/public/notes/{token}` | no | Read-only access via public token. |
| GET | `/tags` | yes | List tags used by the user's notes. |
| GET | `/search?q=` | yes | Full-text search over title, content, tags. |
| GET | `/about` | no | Service metadata. |
| GET | `/health` | no | Liveness probe. |
| GET | `/openapi.json` | no | OpenAPI 3 schema. |
| GET | `/docs/` | no | Swagger UI. |

## Custom feature — public share links

Each note can issue a single unguessable token (`secrets.token_urlsafe(32)`).
`POST /notes/{id}/public-link` creates or rotates it; `DELETE` revokes it.
Anyone holding the token can read the note at `/public/notes/{token}` without
auth. Rotating the token invalidates the previous one, so links can be
revoked without deleting the note.

## Docker quickstart

```bash
cp .env.example .env
docker compose up --build              # api on :8000, postgres on :5432
docker compose run --rm migrate        # apply migrations on demand
```

## Render deploy

`render.yaml` provisions a `web` service plus a free Postgres database
named `notes-db`. Build runs `./build.sh` (install, collectstatic, migrate);
start runs gunicorn with 3 workers and 2 threads. `DJANGO_SECRET_KEY` is
generated by Render; `DATABASE_URL` is wired from the database service.

Set `DJANGO_CSRF_TRUSTED_ORIGINS` and `CORS_ALLOWED_ORIGINS` in the Render
dashboard once the frontend URL is known, and flip `CORS_ALLOW_ALL` to
`False`.
